# Deliverables Desain

Deliverables desain merupakan output nyata yang Anda terima selama dan setelah proyek desain UI/UX. Deliverables ini berfungsi sebagai dokumentasi, panduan implementasi, dan aset yang dapat digunakan untuk pengembangan produk digital Anda. Di agency kami, kami berkomitmen untuk menyediakan deliverables komprehensif, terorganisir dengan baik, dan siap implementasi yang memastikan visi desain Anda direalisasikan dengan tepat. Artikel ini menjelaskan secara detail berbagai deliverables yang akan Anda terima saat bekerja sama dengan kami.

## File Format dan Struktur

Format dan organisasi file yang tepat sangat penting untuk memastikan handoff yang efisien antara tim desain dan pengembangan.

### Format File Desain

Berbagai format file yang kami gunakan dan tujuannya:

**Source Files**

File asli yang menyimpan semua elemen desain:
- Figma files (.fig) sebagai format utama untuk design collaboration
- Adobe XD files (.xd) jika diminta atau sesuai kebutuhan proyek
- Sketch files (.sketch) untuk proyek yang memerlukan kompatibilitas dengan ekosistem Sketch
- Adobe Photoshop files (.psd) untuk manipulasi gambar kompleks
- Adobe Illustrator files (.ai) untuk ilustrasi dan asset vector
- Version history dan organizational structure dalam file
- Shared libraries dan component systems
- Design documentation dalam file comments

**Export Formats**

Format yang dioptimalkan untuk berbagai tujuan:
- PNG dengan transparency untuk UI elements dan icons
- JPG untuk image assets dengan kompresi yang dioptimalkan
- SVG untuk vector graphics, icons, dan illustrations
- PDF untuk documentation dan printable specifications
- WebP untuk web-optimized images dengan quality tinggi
- GIF atau MP4 untuk animation previews
- Lottie files (.json) untuk animations yang complex
- Various resolution exports untuk different device densities (1x, 2x, 3x)

**Prototype Exports**

Format untuk sharing dan reviewing prototypes:
- Interactive HTML prototypes untuk web viewing
- Public sharing links dari Figma, Adobe XD, atau InVision
- Recorded prototype walkthroughs (MP4/MOV)
- Embeddable prototype codes untuk integration dengan documentation
- Offline prototype packages untuk presentations
- Password-protected prototype links untuk confidential projects
- Mobile preview apps links
- QR codes untuk quick access ke prototypes

### Organizational Structure

Bagaimana kami mengatur file untuk accessibility dan maintainability:

**File Naming Conventions**

Sistem penamaan yang konsisten dan informatif:
- Project identifier prefixes
- Version numbers atau dates
- Status indicators (Draft, Review, Final)
- Screen atau feature identifiers
- Sequential numbering jika applicable
- Author initials untuk collaborative projects
- Platform indicators (iOS, Android, Web)
- Purpose labels (Wireframe, Visual, Prototype)

**Folder Hierarchy**

Struktur folder logical untuk navigasi yang mudah:
- Research materials dan insights
- User flows dan information architecture
- Wireframes (low, medium, high fidelity)
- Visual design explorations dan final designs
- Design system dan component library
- Prototypes by version atau feature
- Export assets organized by platform
- Documentation dan specifications
- Archive for outdated versions

**Version Control**

Sistem untuk tracking perubahan dan iterations:
- Semantic versioning (Major.Minor.Patch)
- Date-based versioning untuk regular updates
- Clear version history documentation
- Change logs dengan key updates
- Named iterations untuk major milestones
- Branching structure untuk parallel explorations
- Archive system untuk previous versions
- Restoration points untuk significant stages

**Delivery Methods**

Cara kami menyampaikan file kepada Anda:
- Secure cloud storage platforms (Google Drive, Dropbox)
- Design platform native sharing (Figma, Adobe Cloud)
- Version control systems untuk code-related assets (GitHub, GitLab)
- Encrypted file transfers untuk sensitive projects
- Asset management systems integration jika client uses them
- Direct CMS integration jika applicable
- Regular backup scheduling
- Access permission management

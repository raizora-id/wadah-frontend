# Penghentian Layanan

Meskipun kami berharap untuk menjalin kerjasama jangka panjang, kami memahami bahwa ada saatnya layanan perlu diakhiri. Dokumen ini menjelaskan prosedur penghentian layanan dan langkah-langkah yang diambil untuk memastikan transisi yang lancar dan aman bagi data Anda.

## Prosedur Pengakhiran Kontrak

Kami menawarkan proses pengakhiran yang terstruktur, transparan, dan profesional.

### Inisiasi Pengakhiran

Pengakhiran kontrak dapat diinisiasi melalui beberapa cara:

- **Pemberitahuan Tertulis** - Pengajuan form pengakhiran layanan atau surat resmi
- **Email Formal** - Dikirim dari email terdaftar ke [pengakhiran@namaagency.com](mailto:pengakhiran@namaagency.com)
- **Portal Klien** - Mengisi form pengakhiran melalui dashboard akun Anda

Semua permintaan pengakhiran harus mencakup:
- Nama bisnis dan nomor ID klien
- Layanan spesifik yang akan diakhiri
- Tanggal pengakhiran yang diinginkan
- Alasan pengakhiran (opsional, untuk peningkatan layanan)
- Informasi kontak untuk koordinasi

### Periode Pemberitahuan

Setiap layanan memiliki periode pemberitahuan minimum yang harus dipenuhi:

- **Layanan Website & Aplikasi Mobile** - 30 hari kalender
- **Layanan Desain UI/UX** - Sesuai ketentuan dalam kontrak proyek
- **Paket IT Support** - 60 hari kalender
- **Produk SaaS (Dihadiri & Klola)** - Berlaku hingga akhir periode langganan

Pengakhiran sebelum berakhirnya periode minimum kontrak mungkin dikenakan biaya pengakhiran awal sesuai perjanjian.

### Verifikasi dan Konfirmasi

Setelah permintaan pengakhiran diterima:

1. **Verifikasi Permintaan** - Kami akan memverifikasi bahwa permintaan berasal dari pihak berwenang
2. **Konfirmasi Penerimaan** - Konfirmasi tertulis akan dikirim dalam 2 hari kerja
3. **Assessment Pengakhiran** - Kami akan meninjau status akun, layanan aktif, dan kewajiban kontraktual
4. **Proposal Pengakhiran** - Dokumen detail akan dikirim mencakup:
   - Tanggal efektif pengakhiran
   - Biaya atau kredit yang tersisa
   - Jadwal untuk aktivitas transisi
   - Opsi untuk data dan aset
   - Layanan transisi yang tersedia

### Penyelesaian Kewajiban Finansial

Semua kewajiban finansial harus diselesaikan sebelum pengakhiran final:

- **Invoice Tertunggak** - Harus dilunasi sebelum pengakhiran final
- **Pengembalian Dana** - Jika berlaku, akan diproses dalam 15 hari kerja setelah pengakhiran
- **Biaya Pengakhiran** - Jika ada biaya pengakhiran awal, akan ditagihkan dalam invoice final
- **Asset Transfer Fee** - Biaya transfer aset atau data kompleks jika diperlukan

## Export dan Backup Data

Kami berkomitmen untuk memastikan Anda memiliki akses penuh ke data Anda saat layanan berakhir.

### Data yang Tersedia untuk Export

Bergantung pada layanan yang digunakan, data berikut tersedia untuk diexport:

- **Website**
  - Konten (teks, gambar, video, dokumen)
  - Database situs
  - Konfigurasi dan pengaturan
  - File tema dan template
  - Log dan statistik pengunjung (terbatas)

- **Aplikasi Mobile**
  - Data pengguna
  - Konten aplikasi
  - Konfigurasi backend
  - Aset media

- **Sistem ERP "Klola"**
  - Data transaksional
  - Catatan pelanggan dan vendor
  - Laporan keuangan dan inventaris
  - Dokumen yang diunggah
  - Data historis

- **Undangan Digital "Dihadiri"**
  - Informasi acara
  - Data tamu
  - Aset media
  - Statistik dan tanggapan

### Format dan Metode Export

Kami menyediakan data dalam format standar industri:

- **Format Terstruktur**
  - CSV untuk data tabular
  - JSON atau XML untuk data terstruktur
  - SQL dump untuk database
  - PDF untuk dokumen dan laporan

- **Format Media**
  - Aset media dalam format asli (JPG, PNG, MP4, dll.)
  - Dokumen dalam format asli atau PDF

- **Metode Pengiriman**
  - Portal pengunduhan aman (tersedia selama 14 hari)
  - Transfer SFTP untuk data berukuran besar
  - Hard drive terenkripsi untuk data sangat besar (biaya tambahan mungkin berlaku)
  - API export untuk klien tertentu

### Proses Backup Pre-Terminasi

Sebelum pengakhiran final, kami melakukan langkah-langkah berikut:

1. **Backup Penuh** - Backup komprehensif semua data dan konfigurasi
2. **Verifikasi Integritas** - Pengujian integritas data untuk memastikan kelengkapan
3. **Dokumentasi Sistem** - Inventaris komponen dan dependensi sistem
4. **Kontrol Kualitas** - Verifikasi bahwa data dapat diakses dan digunakan

### Periode Retensi Data Pasca Terminasi

Setelah pengakhiran, data Anda dipertahankan untuk jangka waktu terbatas:

- **Data Lengkap** - Disimpan selama 30 hari setelah pengakhiran
- **Backup Arsip** - Disimpan selama 90 hari (dapat diperpanjang dengan biaya tambahan)
- **Log Sistem** - Disimpan sesuai kebijakan retensi kami (umumnya 1 tahun)
- **Data yang Diwajibkan Secara Hukum** - Disimpan sesuai ketentuan hukum yang berlaku

Setelah periode retensi, data akan dihapus secara permanen menggunakan metode penghapusan aman.

## Transisi ke Sistem Lain

Kami menyediakan dukungan untuk memastikan transisi ke sistem baru berjalan lancar.

### Layanan Transisi

Berbagai tingkat dukungan transisi tersedia:

- **Dukungan Dasar** (Tanpa Biaya)
  - Dokumentasi format data
  - Panduan export data mandiri
  - Checklist migrasi standar
  - Email support terbatas (5 tiket)

- **Dukungan Standard** (Berbayar)
  - Semua fitur dukungan dasar
  - Konsultasi migrasi (2 jam)
  - Bantuan export data khusus
  - Dokumentasi konfigurasi detail
  - Email dan telepon support (10 tiket)

- **Dukungan Premium** (Berbayar)
  - Semua fitur dukungan standard
  - Migrasi data yang dibantu penuh
  - Konfigurasi paralel selama masa transisi
  - Konsultasi arsitektur sistem baru (4 jam)
  - Support prioritas (tidak terbatas selama 30 hari)
  - Personel dedikasi untuk proyek transisi

### Kompatibilitas dan Transfer Pengetahuan

Kami berkomitmen untuk memfasilitasi perpindahan ke platform baru:

- **Dokumentasi API** - Menyediakan dokumentasi API lengkap untuk integrasi
- **Skema Data** - Menjelaskan struktur data untuk memudahkan pemetaan ke sistem baru
- **Workflow Documentation** - Mendokumentasikan proses bisnis dalam sistem kami
- **Knowledge Transfer Session** - Sesi dengan tim teknis untuk mentransfer pengetahuan operasional
- **Konsultasi Vendor** - Koordinasi dengan vendor baru jika diizinkan

### Timeline Transisi Tipikal

Transisi biasanya mengikuti timeline berikut setelah pemberitahuan pengakhiran:

- **Minggu 1-2**: Perencanaan dan persiapan export data
- **Minggu 3-4**: Export dan validasi data
- **Minggu 5-6**: Migrasi data ke sistem baru (jika berlaku)
- **Minggu 7-8**: Pengujian paralel dan verifikasi
- **Minggu 9**: Cutover ke sistem baru
- **Minggu 10-12**: Fase dukungan post-migrasi (jika dipilih)

Timeline aktual dapat bervariasi berdasarkan kompleksitas dan volume data.

### Pertimbangan Keamanan Selama Transisi

Kami menerapkan protokol keamanan ketat selama proses transisi:

- **Enkripsi End-to-End** - Semua data yang ditransfer dienkripsi
- **Akses Terbatas** - Hanya personel yang diberi wewenang yang dapat mengakses data
- **Audit Trail** - Semua akses dan aktivitas selama transisi dicatat
- **Secure Deletion** - Penghapusan aman data dari sistem kami setelah konfirmasi migrasi sukses
- **NDA** - Perjanjian kerahasiaan untuk semua pihak yang terlibat dalam proses transisi

## Penghentian Darurat

Dalam situasi luar biasa, mungkin diperlukan penghentian darurat:

### Kondisi Penghentian Darurat

Beberapa kondisi yang dapat memicu penghentian darurat:

- Pelanggaran material terhadap syarat layanan
- Aktivitas ilegal atau berbahaya
- Kebangkrutan atau likuidasi
- Force majeure (bencana alam, pandemi, dll.)

### Proses Penghentian Darurat

Prosedur khusus diterapkan dalam kasus penghentian darurat:

1. **Notifikasi Segera** - Komunikasi melalui semua saluran yang tersedia
2. **Backup Data Darurat** - Backup komprehensif dilakukan segera
3. **Penawaran Export Data** - Opsi untuk mengekspor data segera
4. **Dukungan Prioritas** - Dukungan prioritas untuk meminimalkan gangguan

## Rekomenkuliasi dan Feedback

Setelah pengakhiran layanan:

- **Exit Survey** - Kami mengundang Anda untuk memberikan feedback tentang layanan dan proses pengakhiran
- **Kemungkinan Rekonsiliasi** - Kami terbuka untuk diskusi jika Anda mempertimbangkan untuk kembali di masa depan
- **Dokumen Penutupan** - Sertifikat penutupan layanan dan konfirmasi penghapusan data (jika diminta)
